### 1.1) `0.72.0` ➜ `0.73.0` (release 2025-12-08)

### 1.5) `0.74.1` ➜ `0.75.0` (release 2026-01-02)

#### 0) Tổng quan
`0.75.0` là minor release nhưng có “khối thay đổi lớn” về tooling upstream:
- ✅ Features UI nhỏ
- ✅ Bugfix
- 🔥 Poetry ➜ uv (chủ yếu ảnh hưởng contributor/build upstream, user pip thường không cần)

#### ✅ Bug Fixes
**1) Fix `navigation_user.html` cho custom user model**
- Quan trọng nếu bạn dùng custom user model (không có `username`)
- Upstream đổi:
```django
{# OLD #}
{{ user.get_full_name|default:user.username }}

{# NEW #}
{{ user.get_full_name|default:user.get_username }}
```
> Nếu bạn override `templates/unfold/helpers/navigation_user.html` ➜ ✅ nhớ merge lại điểm này

**2) Fix version path (semantic-release)**
- semantic-release version_toml: `tool.poetry.version` ➜ `project.version`

#### ✅ Features
**1) Button component thêm relative**
- File: `templates/unfold/components/button.html`
```diff
- class="... px-3 py-2 rounded-default ..."
+ class="... px-3 py-2 relative rounded-default ..."
```

**2) Messages thêm class extra để inject CSS**
- Các file: `templates/unfold/helpers/messages/*.html`
- Thêm:
```django
{% if class %} {{ class }}{% endif %}
```
=> Bạn có thể custom style message theo ngữ cảnh.

**3) Settings form classes (FORMS.classes)**
- Thêm cấu hình mặc định:
  - `FORMS.classes.text_input`
  - `FORMS.classes.checkbox`
  - `FORMS.classes.button`
  - `FORMS.classes.radio`
  - `FORMS.classes.switch`
  - `FORMS.classes.file`
- Và `site.each_context()` sẽ lấy theo config thay vì hardcode trong code.

#### 🧰 Tooling upstream: Poetry ➜ uv
- `.devcontainer` / GitHub Actions / lockfile đổi sang `uv`
- `poetry.lock` bị xóa, thay bằng `uv.lock`
- Nếu bạn chỉ dùng `pip install django-unfold` thì không cần làm gì.

#### 🔧 Internal refactor: filters typing/facets
- Các filter refactor type hints + logic facets (`choices()` trả Iterator / yield)
- Một số filter thêm guard `if facet_counts` để tránh crash

### 2.3) Fieldset tabs: state + error badge (0.76 style)
**File:** `templates/unfold/helpers/fieldsets_tabs.html`

Mẫu logic khuyến nghị:
```django
{% with tabs=adminform|tabs %}
  {% if tabs %}
    {% with active_tab=tabs|tabs_active %}
      <div x-data="{ activeFieldsetTab: '{{ active_tab }}'}">
        <nav>
          {% for fieldset in tabs %}
            <a
              x-on:click="activeFieldsetTab='{{ fieldset.name|slugify }}'"
              x-bind:class="activeFieldsetTab=='{{ fieldset.name|slugify }}' && 'active'"
            >
              {% with error_count=fieldset|tabs_errors_count %}
                {% if error_count > 0 %}
                  <span data-testid="error-count">{{ error_count }}</span>
                {% endif %}
              {% endwith %}
              {{ fieldset.name }}
            </a>
          {% endfor %}
        </nav>

        {% for fieldset in tabs %}
          <div x-show="activeFieldsetTab == '{{ fieldset.name|slugify }}'">
            {% include 'admin/includes/fieldset.html' with stacked=1 %}
          </div>
        {% endfor %}
      </div>
    {% endwith %}
  {% endif %}
{% endwith %}
```
