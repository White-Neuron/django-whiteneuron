# Release v0.3.4.8 (2026-05-24)

## Scope Summary

This release introduces a caching layer for badge callbacks across the dashboard and notification system, significantly reducing database query load by caching badge counts with a 60-second TTL.

## New Changes

### Added
- **Dashboard badge caching**: Django cache-backed badge callbacks for `User`, `UserActivity`, `AnonymousActivity`, and `VisitProfile` models with 60s TTL to reduce repeated database queries on dashboard loads.
- **Notification badge caching**: Per-user cached notification badge with 60s TTL, including an unauthenticated user guard that returns `0`.

### Improved
- Replaced `timezone.now()` with `timezone.localtime()` across all badge callback functions for accurate timezone-aware daily counting.
