## Summary

This release adds a `uuid` field to the custom `User` model, providing a stable unique identifier that is independent of the auto-incrementing primary key. The UUID is automatically generated on user creation and displayed in the Django admin interface for easy reference.

## Main Changes

### Added
- **`whiteneuron/base/models.py`**: New `uuid` field (`UUIDField`, unique, auto-generated via `uuid.uuid4`) on the custom `User` model. The field is read-only at the database level and cannot be manually modified.
- **`whiteneuron/base/admin.py`**: 
  - `uuid` column added to User changelist `list_display`.
  - `uuid` shown as a readonly field in the User change form under "Personal info" section.
  - Excluded from grid view cards to avoid cluttering the visual layout.

### Migration Details
- **`0018_user_uuid`**: Three-step safe migration:
  1. Add nullable `uuid` column (no constraint).
  2. Backfill existing users with unique UUID values via `RunPython`.
  3. Alter field to add `unique=True` and `default=uuid.uuid4`.

## Compatibility Notes

- **Database**: SQLite, PostgreSQL, and MySQL all fully supported. The UUID migration uses Django's standard field alteration which handles schema changes safely on all backends.
- **Existing installations**: The data migration (`RunPython`) ensures all existing users receive a valid UUID before the unique constraint is applied. No manual intervention required.
- **API consumers**: If your API serializes `User` objects, add `uuid` to serializers as needed. It is not included in default admin list display beyond what's configured here.

## Upgrade Guidance

1. Update to `django-whiteneuron==0.3.1.8`:
   ```bash
   pip install --upgrade django-whiteneuron
   # or
   uv upgrade django-whiteneuron
   ```

2. Run migrations:
   ```bash
   python manage.py migrate base
   ```

3. The migration will automatically backfill UUIDs for existing users. For large user tables (10k+ records), this may take a few seconds. Monitor progress via database logs if needed.

## Rollback Note

If you need to rollback, revert the package version and run:
```bash
python manage.py migrate base 0017_alter_uablacklist_pattern
```
This will remove the uuid column. All data is preserved in previous migration states.
