# v0.2.39 — 2026-03-31

## Fix: `get_client_ip()` — validate IP headers, fix block-all bug với Cloudflare Tunnel

### Overview

Bản vá khẩn cho `get_client_ip()` trong `RateLimitMiddleware` và `UserActivityMiddleware`. Hàm này nhận IP raw từ HTTP headers mà không validate format, dẫn đến 2 vấn đề nghiêm trọng: (1) raw string không hợp lệ đi thẳng vào Redis cache key; (2) khi dùng Cloudflare Tunnel, toàn bộ user bị map về cùng 1 IP → block 1 người = block tất cả.

---

### Root Cause

| Vấn đề | Nguyên nhân |
|--------|-------------|
| Header không validate | `request.headers.get("CF-Connecting-IP")` trả về raw string, không qua `ipaddress.ip_address()` |
| Block nhầm tất cả user | Cloudflare **Tunnel** không set `CF-Connecting-IP` → code fall xuống XFF → XFF chứa Cloudflare Edge IP là entry đầu tiên → mọi user share 1 bucket |
| `REMOTE_ADDR` không validate | Raw value từ `request.META` đi thẳng vào cache key |

---

### Changes

#### `_parse_ip(raw)` — helper mới

```python
def _parse_ip(raw: str | None) -> str | None:
    """Validate và trả về IP string chuẩn hóa, None nếu không hợp lệ."""
    if not raw:
        return None
    raw = raw.strip()
    try:
        return str(ipaddress.ip_address(raw))
    except ValueError:
        return None
```

- Normalize về canonical form (`::1` thay vì `0:0:0:0:0:0:0:1`)
- Trả `None` nếu không hợp lệ thay vì pass raw string
- Dùng cho tất cả sources: CF headers, XFF entries, REMOTE_ADDR

#### `get_client_ip()` — refactored

**Trước:**
```python
ip = request.headers.get("CF-Connecting-IP")  # raw string, không validate
if ip:
    return ip, ...
```

**Sau:**
```python
ip = _parse_ip(request.headers.get("CF-Connecting-IP"))  # validate trước
if ip:
    return ip, ...
```

XFF loop cũng refactored từ list comprehension sang explicit loop với `_parse_ip()` per entry.

#### Comment cảnh báo Cloudflare Tunnel vs Proxy

Thêm comment rõ ràng:
```
# CF-Connecting-IP: chỉ tin tưởng khi deploy sau Cloudflare Proxy (không phải Tunnel).
# Cloudflare Tunnel (cloudflared) KHÔNG set header này — nếu dùng Tunnel, tắt cờ này.
```
