# Release v0.3.4.10 — Security patch: Upgrade Django to 6.0.6+ (CVE-2026-8404, CVE-2026-48587, CVE-2026-6873)

## 1. Release Summary
- **Version**: v0.3.4.10 (2026-06-12)
- **Release objective**: Patch 5 Django security vulnerabilities by upgrading django lower bound and pinning upper bound to prevent unexpected major upgrades
- **Scope of changes**: Dependency update in pyproject.toml, version bump

## 2. Main Changes
- **Fixed**: Upgraded `django` dependency from `>=6.0.6` to `>=6.0.6,<7.0.0` — patches the following CVEs:
  - **CVE-2026-8404** (CVSS 5.3 Medium) — Django security vulnerability fixed in 6.0.5+
  - **CVE-2026-48587** (CVSS 5.3 Medium) — Django security vulnerability fixed in 6.0.5+
  - **CVE-2026-6873** (CVSS 4.3 Medium) — Django security vulnerability fixed in 6.0.5+
  - **CVE-2026-7666** (CVSS 2.3 Low) — Django security vulnerability fixed in 6.0.5+
  - **CVE-2026-35193** (CVSS 2.3 Low) — Django security vulnerability fixed in 6.0.5+
- **Improved**: Added upper bound `<7.0.0` to prevent unexpected major version upgrades that may introduce breaking changes

## 3. Compatibility & Risk
- **Compatibility assessment**: No breaking changes; safe for all v0.3.x users
- **Remaining risk**: Low — Django 6.0.x is a minor release within the same major version; upper bound prevents accidental upgrade to Django 7.x
- **Mitigation plan**: Pin django <7.0.0 ensures controlled upgrade path; rollback to v0.3.4.9 if issues arise
