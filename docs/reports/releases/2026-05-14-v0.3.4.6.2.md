# Release v0.3.4.6.2 — 2026-05-14

## Summary

Configuration improvement: Add support for configurable rate-limit exempt paths and user activity exclude paths via Django settings. This allows deployments to customize which paths should be excluded from rate limiting and activity tracking without modifying source code.

## Main Changes

### Improved
- **RateLimitMiddleware**: Now reads `RATE_LIMIT_EXEMPT_PATHS` from Django settings — a comma-separated list of additional path prefixes to exclude from rate limiting. Paths are loaded dynamically at middleware initialization, allowing runtime configuration changes.
- **UserActivityMiddleware**: Now reads `USER_ACTIVITY_EXCLUDE_PATHS` from Django settings — a semicolon-separated list of `path,condition` pairs for excluding paths from activity tracking and rate limiting. Supports three conditions: `startwith`, `contains`, `exact`. Invalid entries are safely skipped.

## Compatibility Notes

- **No breaking changes** — all new settings are optional. Existing deployments work without any configuration changes.
- Safe for all v0.3.x users.
- New settings (`RATE_LIMIT_EXEMPT_PATHS`, `USER_ACTIVITY_EXCLUDE_PATHS`) must be defined in Django settings to take effect.

## Upgrade Guidance

1. Upgrade the package:
   ```bash
   pip install --upgrade django-whiteneuron
   ```
   Or with uv:
   ```bash
   uv add "django-whiteneuron>=0.3.4.6.2"
   ```

2. (Optional) Configure path exclusions in Django settings:
   ```python
   # Exclude specific paths from rate limiting
   RATE_LIMIT_EXEMPT_PATHS = "/health,/ping,/ws/"

   # Exclude paths from activity tracking (format: "path,condition;path,condition")
   USER_ACTIVITY_EXCLUDE_PATHS = "/static,startwith;/health,startwith"
   ```

## Rollback

Safe to revert to v0.3.4.6.1 if needed. No schema changes introduced — rollback requires no database migration steps.
