# Release v0.3.4.9 — Bugfix: startswith typo in UserActivityMiddleware

## 1. Release Summary
- **Version**: v0.3.4.9 (2026-05-27)
- **Release objective**: Fix critical bug where `/health/` endpoint was still being tracked as anonymous activity due to typo `startwith` → `startswith` in UserActivityMiddleware
- **Scope of changes**: 1 source file fix, documentation corrections

## 2. Main Changes
- **Fixed**: Typo `'startwith'` → `'startswith'` in two places within `UserActivityMiddleware`:
  - Line 268: Condition validation in `__init__()` — prevents custom exclude paths from being silently skipped
  - Line 283: Path matching check in `do_not_track()` — restores `/health/`, `/ws/` and all custom path exclusions from activity tracking and rate limiting
- **Fixed**: Documentation updated to use correct condition name `startswith` instead of typo `startwith`

## 3. Compatibility & Risk
- **Compatibility assessment**: No breaking changes; safe for all v0.3.x users
- **Remaining risk**: None — this is a pure bugfix with no schema or API changes
- **Mitigation plan**: N/A — rollback is straightforward if needed
