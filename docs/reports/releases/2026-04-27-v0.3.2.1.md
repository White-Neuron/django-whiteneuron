# Release v0.3.2.1 — JSON Payload Capture Fix in UserActivityMiddleware

## 1. Summary
- **Version**: `v0.3.2.1`
- **Release Date**: 2026-04-27
- **Scope**: Bug fix for request payload logging in activity tracking middleware

## 2. Main Changes
- **Request data capture enhanced** — `UserActivityMiddleware` now captures both form-encoded (`application/x-www-form-urlencoded`) and JSON (`application/json`) request payloads via new `_get_request_data()` helper method. Previously, only `request.POST` was captured, silently dropping JSON bodies sent by API clients.

## 3. Fixed
- **JSON payload not logged** — `UserActivityMiddleware.__call__()` called `self._sanitize_post(request.POST.dict())` directly, which returns empty dict for JSON requests since Django does not parse JSON into `request.POST`. Added `_get_request_data()` that checks `request.POST` first (form data), then falls back to parsing `request.body` as JSON when POST is empty. Sensitive-field sanitization (`_sanitize_post`) is applied to both code paths.

## 4. Compatibility & Risk
- **Compatibility**: Django >= 5.2.13, Python >= 3.11 — no breaking changes, purely additive fix.
- **Risk**: Negligible. Only affects the activity logging middleware; existing `UserActivity` and `AnonymousActivity` records are unaffected. No migrations required.
- **Mitigation**: None needed — drop-in replacement with identical interface.
