# v0.2.44 — 2026-04-02

## Fixes & Improvements: Feedback System — Error handling, input limits, i18n lazy evaluation

### Overview

Vá các lỗi còn lại trong hệ thống Feedback và Admin sau v0.2.43: fix HTTP 403 khi resolve feedback, fix HTTP 414 khi note quá dài, thêm giới hạn ký tự hai chiều (client + server), cải thiện xử lý lỗi chi tiết theo status code, và sửa `search_help_text` không phản ứng với đổi ngôn ngữ.

---

### Changes

#### `feedbacks/admin.py` — Resolve feedback flow

- **Fix**: `changeform_view` sau khi resolve thành công trả về `JsonResponse({"success": True})` thay vì tiếp tục gọi `super().changeform_view()` với POST request. Django admin từ chối POST do `has_change_permission = False`, gây HTTP 403 giả — feedback đã lưu nhưng JS nhận lỗi.
- **Fix**: Note đọc từ `request.POST.get('note')` thay cho `request.GET.get('note')` — tránh HTTP 414 (URI Too Long) khi note dài.
- **Fix**: Các case lỗi (permission denied, not found, note quá dài) trả về `JsonResponse` có `status` và `message` thay vì fallback `super().changeform_view()` — JS có thể hiển thị đúng thông báo.
- **Added**: Server-side validation: `len(note) > 500` → HTTP 400.
- **Added**: `get_short_message()` trong `list_display`: hiển thị tối đa 200 từ bằng `Truncator` — tránh tràn giao diện danh sách.

#### `feedbacks/urls.py` — Input validation

- **Added**: Server-side validation: `len(feedback_message) > 2000` → HTTP 400 với message rõ ràng.

#### `templates/admin/feedbacks/feedback_change_form.html` — Resolve modal

- **Fix**: `fetch` chuyển sang POST với `FormData` — `note` gửi trong body thay vì URL query string.
- **Fix**: CSRF token lấy từ cookie và đưa vào `FormData`.
- **Improved**: Xử lý lỗi chi tiết theo status code:
  - 403 → "Permission denied. Only superusers can perform this action."
  - 404 → "Feedback not found."
  - 500 → "Server error. Please try again later."
  - Network error → "Network error. Please check your connection."
  - Lỗi khác → "Unexpected error (HTTP {status})."
  - Ưu tiên đọc `data.message` từ JSON response trước khi fallback về message theo status.
- **Improved**: Thông báo lỗi hiển thị màu `text-error`, thành công `text-success`.
- **Improved**: Các nút được re-enable khi có lỗi để user thử lại.
- **Added**: `maxlength="500"` trên input note + counter ký tự động (0/500).
- **Added**: Counter reset về 0 khi mở lại modal.

#### `templates/admin/submit_line.html` — Feedback submit modal

- **Added**: `maxlength="2000"` trên textarea nội dung feedback + counter ký tự động (0/2000).
- **Added**: Guard trong JS: không submit nếu `feedbackMessage.length > 2000`.
- **Added**: Counter reset khi mở lại modal.

#### `base/modeladmin.py` — i18n lazy evaluation

- **Fix**: `search_help_text` chuyển từ `__init__` (evaluate một lần lúc server start) → `@property` (evaluate per-request). Trước đây dùng `_('Search by %s') % fields` — toán tử `%` ép lazy string evaluate ngay tại `__init__`, cứng ngôn ngữ suốt vòng đời process.
- **Fix**: Dùng `format_lazy(_('Search by {fields}'), fields=fields)` — template dịch giữ lazy đến khi render request, phản ứng đúng khi đổi ngôn ngữ.
- **Fix**: `str(get_verbose_name_field(...))` trong property — đảm bảo verbose_name lazy cũng được resolve theo ngôn ngữ hiện tại.

#### `locale/vi/LC_MESSAGES/django.po`

- Cập nhật msgid `'Search by %s'` → `'Search by {fields}'`, msgstr `'Tìm kiếm theo {fields}'`.
- Compile lại file `.mo`.

#### `base/utils.py` + `feedbacks/utils.py` — Badge callback

- **Improved**: `base_badge_callback()` accepts optional `filter_kwargs` parameter — allows callers to pass additional queryset filters without subclassing.
- **Improved**: `feedback_data_badge_callback()` now scopes the badge count to the current user for non-superusers (`filter_kwargs = {'user': request.user}`); superusers continue to see the global count.

### Compatibility & Risk

- Không có migration mới.
- Không thay đổi public API.
- Thay đổi `search_help_text` từ instance attribute sang `@property` — tương thích với UnfoldAdmin và Django admin; không ảnh hưởng subclass nào trừ khi subclass tự set `self.search_help_text` trong `__init__` (property sẽ override).
