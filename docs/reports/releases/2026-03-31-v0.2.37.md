# Release v0.2.37 — 2026-03-31

## 1. Tóm Tắt Phát Hành

- **Phiên bản:** 0.2.37
- **Mục tiêu phát hành:** Bảo mật guest login — xoá password hardcode công khai trong HTML, thay bằng `GuestLoginView` server-side POST-only có CSRF protection; cập nhật GitHub Actions sang Node.js 24.
- **Phạm vi thay đổi:** `base/management/commands/init_guest.py`, `base/views.py`, `base/urls.py`, `templates/admin/login_after.html`, `.github/workflows/python-publish.yml` — không có migration mới.

## 2. Thay Đổi Chính

- **Security — Xoá hardcode password guest:** Password `'whiteneuron-guest-2024@'` trong `init_guest.py` được thay bằng `set_unusable_password()`. Guest user không còn có password hợp lệ — không thể đăng nhập bằng credentials, không thể bị bruteforce.
- **Security — Xoá JS fill-password trong HTML:** Button "Log in as guest" trước đây dùng JavaScript điền `id_password` bằng password plaintext rồi submit form — bất kỳ ai xem source trang login đều thấy. Toàn bộ đoạn JavaScript này đã bị xoá.
- **Added — `GuestLoginView`:** View POST-only tại `base/guest-login/` — gọi `login()` trực tiếp không qua password check, validate `next` redirect bằng `url_has_allowed_host_and_scheme()` chống open redirect, có CSRF protection qua `{% csrf_token %}` trong template.
- **Improved — `init_guest` command:** Thêm `is_staff=True` khi tạo user mới (cần thiết để `AutoGuestLoginMiddleware` nhận diện guest hợp lệ theo filter `is_staff=True`).
- **CI — Node.js 24:** Thêm `FORCE_JAVASCRIPT_ACTIONS_TO_NODE24: true` vào workflow `python-publish.yml` — opt-in chính thức vào Node 24 cho GitHub Actions runner, tránh deprecation warning trước deadline 2/6/2026.

## 3. Phân Tích Bảo Mật

**Trước v0.2.37:**
```html
<!-- Hiển thị công khai trong source HTML trang login -->
<script>
    document.getElementById("id_password").value = "whiteneuron-guest-2024@";
    document.getElementById("login-form").submit();
</script>
```
→ Password guest public hoàn toàn. Bất kỳ ai cũng đọc được, script có thể bị crawl, cache, index.

**Sau v0.2.37:**
```html
<!-- Chỉ là form POST thông thường, không có credential nào -->
<form method="post" action="{% url 'guest_login' %}">
    {% csrf_token %}
    <button type="submit">Log in as guest</button>
</form>
```
→ Không có credential trong HTML. Server xác thực danh tính guest qua `username='guest'` + `is_active=True` + `is_staff=True`.

## 4. Tương Thích Và Rủi Ro

- **Không breaking change** — `AutoGuestLoginMiddleware` không thay đổi, vẫn hoạt động bình thường.
- **Migration cần thực hiện trên existing deployment:** Guest user cũ vẫn có usable password. Chạy lại `init_guest` **không** reset password user đã tồn tại (branch `else` chỉ add group). Cần chạy tay hoặc qua Django shell:
  ```python
  from whiteneuron.base.models import User
  u = User.objects.get(username='guest')
  u.set_unusable_password()
  u.save(update_fields=['password'])
  ```
- Không có migration DB mới.
