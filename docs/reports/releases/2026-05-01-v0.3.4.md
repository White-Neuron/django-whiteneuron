# Release Notes - v0.3.4 (2026-05-01)

## Summary
This release focuses on enhancing the reliability and security of user activity tracking, specifically improving how JSON payloads are captured in middleware and introducing configurable anti-spam measures for the feedback system.

## Main Changes

### Added
- **Configurable Feedback Cooldown**: Introduced `FEEDBACK_COOLDOWN_SECONDS` setting to allow administrators to control the anti-spam cooldown period for feedback submissions.

### Improved
- **Enhanced JSON Activity Logging**: `UserActivityMiddleware` now correctly captures and sanitizes JSON request bodies (POST, PATCH, PUT, DELETE), ensuring API interactions are accurately recorded in activity logs.
- **Request Body Size Limit**: Implemented a 1MB limit on JSON request bodies within the middleware to mitigate potential Denial of Service (DoS) attacks via large payloads.
- **Expanded HTTP Method Support**: `UserActivity` and `AnonymousActivity` models now support all standard HTTP methods (`GET`, `POST`, `PUT`, `PATCH`, `DELETE`, `HEAD`, `OPTIONS`).
- **Consistent Admin Badges**: Updated badge callbacks for user, anonymous, and visit profile activities to return `0` instead of an empty string when no data is present, ensuring stable UI rendering in the admin dashboard.

### Fixed
- **Feedback Cooldown Integration**: Ensured the feedback system correctly utilizes the new `FEEDBACK_COOLDOWN_SECONDS` setting from Django configuration.

## Compatibility & Risk Notes
- **Compatibility**: No breaking changes. This release is backward compatible with all v0.3.x versions.
- **Risk Level**: Low. All changes are additive or involve non-breaking logic improvements.

## Upgrade Guidance
- No manual database migrations are required for this release.
- It is recommended to update to v0.3.4 to benefit from improved activity logging and security hardening.

## Rollback Note
