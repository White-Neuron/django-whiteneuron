# Release v0.3.2.2 — 2026-04-29

## Summary

Patch release focused on access-control hardening for admin actions, a new superuser-only `duplicate_objects` bulk action auto-injected into all `ModelAdmin` subclasses, and a significant expansion of the Vietnamese translation coverage to include upstream `django-unfold` UI strings.

---

## Changes

### Fixed

- **`ModelAdmin.get_actions()` — soft-delete restricted to superusers only**
  Previously, the `delete_selected` action was replaced with a soft-delete handler for _all_ authenticated users. This meant regular staff with delete permissions would see a "delete" action that silently soft-deleted records instead of the expected hard delete — or no feedback at all. The action is now injected only inside the `is_superuser` guard, consistent with `hard_delete` and `restore`.

- **`get_actions()` logic restructured**
  All destructive and recovery actions (`delete_selected` / soft_delete, `hard_delete`, `restore`, `duplicate_objects`) are now gated behind `request.user.is_superuser`, preventing accidental exposure to non-superuser staff accounts.

### Added

- **`duplicate_objects` bulk action** (`whiteneuron/base/modeladmin.py`)
  Automatically injected for superusers into every `ModelAdmin` subclass. Creates copies of all selected objects by nulling the PK and appending ` (Copy)` to the `name` field. Requires `add` permission (`permissions=['add']`). Displays a success message with the count of duplicated objects.

- **`scripts/makemessages.sh` — unfold string extraction via symlink**
  A temporary symlink `whiteneuron/_unfold_src → .venv/lib/python3.11/site-packages/unfold` is created before `makemessages`, enabling `--symlinks` traversal. This allows Django's `makemessages` command to extract translatable strings from the upstream `django-unfold` package without modifying it. The symlink is removed automatically after extraction.

### Improved

- **Vietnamese translation (`locale/vi/LC_MESSAGES/django.po`)** expanded from ~200 entries to ~1,300+ entries, now covering:
  - Unfold core UI: filter labels, action bar, submit line, pagination, search
  - Unfold contrib: import/export workflow, guardian permissions, simple_history views, array forms, constance admin
  - Unfold widgets: date/time pickers, clearable file input, related widget wrapper, currency select
  - All existing project-specific strings retained

## Compatibility Notes

- No schema changes. No migrations added.
- No new dependencies.
- Fully backward-compatible with `0.3.2.1`.
- The `duplicate_objects` action assumes a `name` field on the model. If the model does not have `name`, the action will raise an `AttributeError`. Override `duplicate_objects` in subclasses if needed.

---

## Upgrade Guide

```bash
uv add "django-whiteneuron==0.3.2.2"
```

No configuration changes required. The `duplicate_objects` action is injected automatically — no opt-in needed.

---

## Rollback

Downgrade to `0.3.2.1` if any issue arises:

```bash
uv add "django-whiteneuron==0.3.2.1"
```

No data migration to reverse.
