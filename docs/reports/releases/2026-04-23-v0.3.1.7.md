# Release v0.3.1.7 — 2026-04-23

## 1. Release Summary
- **Version**: 0.3.1.7
- **Release goal**: Ship three independent fixes: a new `post_save` signal that ensures every new `User` receives a login email regardless of creation path; a Django 6 compatibility fix for `format_html`; and re-activating the `[tool.uv.sources]` block so `md2html-tailwind4 >=1.5.0` resolves correctly from GitHub.
- **Scope**: `whiteneuron/base/models.py`, `whiteneuron/base/admin.py`, `pyproject.toml`, `uv.lock`.

## 2. Main Changes

### Added
- **`whiteneuron/base/models.py`** — Added `post_save` import (`post_save` from `django.db.models.signals`) and handler `_send_email_to_new_user`.
  - Fires on every new `User` creation (admin form, API, management commands, Django shell, etc.).
  - Generates a random password via `make_random_password()`, calls `set_password`, saves only the password field, then sends the login email via `send_email_login`.
  - Skips silently when `instance.email` is blank.
  - Connected with `dispatch_uid='base.models._send_email_to_new_user'` to prevent duplicate connections.
- **`whiteneuron/base/admin.py`** — `UserAdmin.save_model` now sets `obj._skip_new_user_email = True` before saving, so the signal skips the instance when the admin form already handled email delivery. This prevents double email sends.

### Fixed
- **`whiteneuron/base/admin.py`** — `MailAdmin.preview_email` changed from `format_html(obj.content)` to `mark_safe(obj.content)`. Django 6.0 enforces that `format_html` is called with at least one positional or keyword argument; passing a bare string raises `TypeError: args or kwargs must be provided.`
- **`pyproject.toml`** — Uncommented `[tool.uv.sources]` block so `md2html-tailwind4` is resolved from the GitHub repository at tag `v1.5.0` rather than PyPI (PyPI only publishes `<=1.2.0`). Without this, `uv sync` fails for Python >= 3.14 / Windows splits.

## 3. New Features
- Login email delivery is now **path-agnostic**: any code that creates a `User` (including third-party apps, management commands, or programmatic creation) will trigger the welcome email automatically, without requiring callers to import or invoke `send_email_login` explicitly.

## 5. Compatibility and Risk Notes
- **No breaking changes.** All changes are additive or fix incorrect behaviour.
- **Django 6 compatibility**: `mark_safe` is the correct API for pre-built trusted HTML strings; `format_html` without placeholders has been invalid since Django 6.
- **Signal idempotency**: `dispatch_uid` prevents duplicate handler registration when the module is imported multiple times.
- **Email guard**: If `INIT_ADMIN_EMAIL`/email is missing, the signal exits cleanly — no exception raised.
- **Admin form path unchanged**: The `save_model` logic in `UserAdmin` is untouched; the only addition is the `_skip_new_user_email` flag before `super().save_model()`.

## 7. Upgrade Guide
- `pip install --upgrade django-whiteneuron` (or `uv add django-whiteneuron>=0.3.1.7`).
- No migrations required.
- No settings changes required.
- If your project creates users programmatically and you do **not** want the new email signal to fire, set `instance._skip_new_user_email = True` before saving the new user.

## 8. Rollback Note
