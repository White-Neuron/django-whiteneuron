# Release Notes — v0.3.4.1

**Date**: 2026-05-02  
**Version**: 0.3.4.1  
**Branch**: main2.0  

## Summary

Patch release focused on security hardening of the `UserActivityMiddleware` payload sanitization and a new management utility for VisitProfile data integrity. No schema changes; safe upgrade for all v0.3.x users.

## Main Changes

### Added
- **Management command** `update_visit_profiles` (`whiteneuron/base/management/commands/update_visit_profiles.py`) — recalculates `first_seen` and `last_seen` timestamps on `VisitProfile` records by aggregating from linked `UserActivity` and `AnonymousActivity` entries. Supports `--dry-run` for preview mode.

### Improved
- **Sensitive value detection** in `_sanitize_post()` (`whiteneuron/base/middleware.py`) — expanded beyond field-name matching to include pattern-based detection:
  - Authorization header values (Bearer tokens, API keys, JWT, session IDs)
  - JWT token format validation (three base64url segments separated by dots)
  - Long base64 strings (≥40 chars with standard alphabet)
  - Long hex strings (≥32 chars)
