## 3. New Features / Improvements

- **Dependency**: `md2html-tailwind4` updated from v1.4.2 to v1.5.0, picking up upstream improvements and the `pymdown-extensions` integration.
- **Build quality**: The `@source` directive ensures Tailwind does not tree-shake classes emitted by `md2html_tailwind4` ā€” previous versions relied on implicit content scanning which could silently miss Python-defined classes.

---

## 4. Compatibility & Risk Notes

- **Backward compatible**: No breaking changes to public APIs, models, or settings.
- **New transitive dependency**: `pymdown-extensions` v10.21.2 is now pulled in transitively via `md2html-tailwind4`. It is a pure-Python library widely used in the Django/Python ecosystem; no known conflicts expected.
- **Tailwind source path**: The added `@source` directive uses the `.venv/lib/python3.11/` path. Projects using a different Python minor version or a non-`.venv` virtual environment will need to adjust this path in their local `styles.css`.
- **Risk level**: Low ā€” no model or logic changes.
