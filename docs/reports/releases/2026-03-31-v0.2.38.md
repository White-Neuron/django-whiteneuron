# v0.2.38 — 2026-03-31

## Security: UA Blacklist — block bots/crawlers by User-Agent

### Overview

Bổ sung tầng bảo vệ thứ hai sau IP Blacklist: block request ngay tại middleware dựa trên `User-Agent` header. Đặc biệt hiệu quả với các AI crawler như GPTBot, ClaudeBot, CCBot vốn không cố định IP nhưng luôn khai báo User-Agent đặc trưng.

---

### Changes

#### Model: `UABlacklist` (`base/ua_blacklists`)

Mô hình mới quản lý danh sách UA bị chặn động qua admin:

| Field | Type | Mô tả |
|-------|------|--------|
| `pattern` | CharField (500) | Keyword hoặc regex để match với `User-Agent` |
| `is_regex` | BooleanField | Nếu tích: dùng `re.search()` case-insensitive; nếu không: substring match |
| `reason` | CharField (500) | Ghi chú lý do chặn |
| `is_active` | BooleanField | Bật/tắt pattern, sync cache ngay khi save |
| `created_at` | DateTimeField | Auto |
| `created_by` | FK → User | Auto gán khi tạo qua admin |

- `rebuild_cache()`: classmethod rebuild `blacklist:ua_patterns` trong Redis sau mỗi save/delete.

#### Admin: `UABlacklistAdmin`

- List display: `pattern`, `is_regex`, active badge, `reason`, `created_at`, `created_by`
- Filters: `is_active`, `is_regex`
- Actions: **Activate / Deactivate** (rebuild Redis cache real-time, không cần restart server)
- Superuser only (implicit via site registration trên `base_admin_site`)

#### Middleware: `RateLimitMiddleware._is_ua_blacklisted()`

Thứ tự check:

1. **Static** — `UA_BLACKLIST` env (comma-separated keywords, loaded at startup, O(n) substring match)
2. **Dynamic** — Redis cache `blacklist:ua_patterns`:
   - Cache hit: sử dụng list patterns từ Redis
   - **Cache miss** (Redis restart / first boot): fallback query DB → warm cache → tiếp tục check
3. Mỗi pattern `is_regex=True`: `re.search(pattern, ua, re.IGNORECASE)`, lỗi regex được skip riêng lẻ thay vì short-circuit toàn bộ vòng lặp

Block flow trong `__call__`: IP check → **UA check** → Rate limit check

#### Config: `env.example`

```env
# Danh sách User-Agent keyword bị chặn tĩnh (comma-separated, khớp substring, không phân biệt hoa/thường)
# Ví dụ: UA_BLACKLIST=GPTBot,ClaudeBot,https://openai.com
# Để chặn động (regex, bật/tắt real-time), dùng UA Blacklist trong admin.
UA_BLACKLIST=
```

### Usage

**Chặn tĩnh qua `.env`** (restart required):
```
UA_BLACKLIST=GPTBot,ClaudeBot,CCBot,https://openai.com
```

**Chặn động qua admin** (real-time, không restart):

Admin → System → UA Blacklist → Add:
- Pattern: `GPTBot` | is_regex: No → block mọi UA chứa "gptbot"
- Pattern: `https://openai\.com/gptbot` | is_regex: Yes → block chính xác theo regex
- Pattern: `(GPTBot|ChatGPT-User|OAI-SearchBot)` | is_regex: Yes → block nhiều bot OpenAI cùng lúc
