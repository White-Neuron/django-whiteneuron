# Release v0.2.26

Date: 2026-03-23

## Scope
- Improve admin loading overlay behavior to handle file downloads without getting stuck.
- Add backend-frontend handshake for download responses.

## Changes
- Added `DownloadResponseFlagMiddleware` to mark attachment responses for frontend loading completion.
- Wired middleware in Django `MIDDLEWARE` stack.
- Updated loading overlay logic in frontend:
  - Use token-based handshake for both internal link clicks and form submits.
  - Add polling timeout guard to avoid infinite loading states.
  - Reset loading/cookies cleanly on `pageshow` and lifecycle transitions.
- Cleaned admin base overlay markup and loading CSS defaults.

## Compatibility and Risk
- Backward compatible for existing admin pages.
- Low risk: changes are limited to loading UX and middleware cookie signaling.
- Potential edge cases: endpoints returning downloadable data without `Content-Disposition: attachment` may require endpoint-specific fallback.

## Validation
- Build completed successfully via `bash scripts/build.sh`.
- Artifacts generated:
  - `dist/django_whiteneuron-0.2.26-py3-none-any.whl`
  - `dist/django_whiteneuron-0.2.26.tar.gz`
- `twine check` passed for wheel and sdist.
- Isolated install/import smoke test passed in temporary virtual environment.

## Upgrade Guidance
- Upgrade package to `0.2.26` and redeploy static assets.
- Clear browser cache or force-refresh admin pages to load updated JS/CSS.

## Rollback
- Roll back to tag `v0.2.25` if regressions are observed.
