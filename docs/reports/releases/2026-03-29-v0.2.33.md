# Release v0.2.33 — 2026-03-29

## 1. Tóm Tắt Phát Hành

- **Phiên bản:** 0.2.33
- **Mục tiêu phát hành:** Fix rate limiting không hoạt động trong môi trường Docker + Daphne, hạ ngưỡng phù hợp với admin panel.
- **Phạm vi thay đổi:** `middleware.py`, `settings.py`, `env.example`, `models.py`, `README.md` — không có migration mới.

## 2. Thay Đổi Chính

- **Fixed — Cache ConnectionError:** `_is_rate_limited()` và `_is_user_rate_limited()` thêm `except Exception: return False`. Trước đây khi Redis unreachable (sai host trong Docker: `127.0.0.1` thay vì tên service), `cache.incr()` ném `ConnectionError` không được bắt → rate limit hoàn toàn vô hiệu.
- **Fixed — Default ngưỡng quá cao:** Hạ `RATE_LIMIT_REQUESTS` 300→60, `USER_RATE_LIMIT_REQUESTS` 200→60 req/60s. Với 300 req/60s, bot có thể spam 299 req/min mãi mãi mà không bị chặn.
- **Fixed — `User.display_header()` hardcode URL:** Thay `/admin/base/user/{id}/change/` bằng `reverse('admin:base_user_change', args=[self.id])` — tránh broken link khi đổi `ADMIN_URL` prefix.
- **Fixed — Default `BEHIND_CLOUDFLARE=True`:** Phù hợp với hạ tầng luôn dùng Cloudflare Tunnel.

## 3. Nguyên Nhân Gốc Rễ (Tại Sao Không Chặn Spam)

```
Docker container → Redis tại 127.0.0.1:6379 → không kết nối được
→ cache.incr() ném ConnectionError
→ except ValueError không bắt được
→ exception propagate → request bypass rate limit
```

Fix: Docker `.env` cần:
```env
CACHE_REDIS_LOCATION=redis://redis:6379   # tên service Docker Compose
RATE_LIMIT_REQUESTS=60
USER_RATE_LIMIT_REQUESTS=60
BEHIND_CLOUDFLARE=True
```
