# Release v0.3.4.6.1 — 2026-05-11

## Summary

Dependency update: Bump `django-unfold` to >=0.92.0 and raise minimum Python version to 3.12+. This release aligns with django-unfold's latest requirements while maintaining backward compatibility for all v0.3.x users on supported Python versions.

## Main Changes

### Updated
- **django-unfold**: Minimum version bumped from `>=0.89.0` to `>=0.92.0` — incorporates improvements from 0.90.0–0.92.0 including compressed fields by default, removal of unbound template blocks, inline before/after templates, and UI component enhancements.
- **Python requirement**: `requires-python` raised from `>=3.11` to `>=3.12` — aligns with django-unfold 0.92+ requirements and modern Python support.

## Files Changed

| File | Change |
|---|---|
| `pyproject.toml` | Version bump (0.3.4.6 → 0.3.4.6.1), django-unfold >=0.92.0, requires-python >=3.12 |
| `uv.lock` | Lock file refreshed for dependency changes |
| `whiteneuron/static/base/css/styles.css` | Tailwind CSS rebuilt with updated dependencies |

## Compatibility Notes

- **Breaking**: Python 3.11 is no longer supported starting from this version. Users on Python 3.11 should remain on v0.3.4.6 or earlier.
- No breaking changes to API, models, admin configuration, or templates.
- Safe for all v0.3.x users running Python 3.12+.

## Upgrade Guidance

1. Ensure your environment uses **Python 3.12** or later:
   ```bash
   python --version  # Should show 3.12.x or higher
   ```
2. Upgrade the package:
   ```bash
   pip install --upgrade django-whiteneuron
   ```
   Or with uv:
   ```bash
   uv add "django-whiteneuron>=0.3.4.6.1"
   ```

## Rollback

Safe to revert to v0.3.4.6 if needed. Downgrade requires Python 3.11 compatibility — ensure your environment supports the earlier version before rolling back.
