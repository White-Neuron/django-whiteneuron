# Release v0.3.0 — Bảo Vệ Toàn Vẹn File với SHA-256

**Ngày phát hành:** 2026-04-13
**Phiên bản:** 0.3.0 (từ 0.2.48.5)
**Nhánh:** main2.0

---

## 1. Tóm Tắt Phát Hành

- **Phiên bản:** 0.2.48.5 → 0.3.0
- **Mục tiêu phát hành:** Tính năng lớn mới — bảo vệ toàn vẹn file (file integrity protection) sử dụng SHA-256 hash cho `ExcelFile` và `PDFFile`.
- **Phạm vi thay đổi:** Module `file_management` hoàn toàn mới: model, view, admin UI, URL routing, widget tùy chỉnh, bản dịch tiếng Việt đầy đủ.

---

## 2. Thay Đổi Chính

### Thêm Mới
- **`compute_file_hash(file_field)`**: Tính SHA-256 hash cho file. Dùng `storage.open()` trực tiếp cho file đã commit (tránh lỗi `FieldFile` state với Celery tasks).
- **`BaseFile.save()` override**: Tự động tính hash khi upload file mới. Reset `status='done'` khi file mới được upload đè lên file lỗi.
- **`BaseFile.verify_integrity()`**: So sánh hash hiện tại với hash lưu trữ, trả về `True/False`.
- **`BaseFile.delete()`**: Dùng `file.delete(save=False)` để xóa file khỏi storage khi xóa record.
- **`download_file` view** (`file_management/views.py`): Endpoint download có xác thực (`@login_required`). Logic:
  - Hash = NULL → tự động enroll (tính và lưu hash lần đầu cho legacy records), rồi serve.
  - Hash không khớp → đánh dấu `status='error'`, trả 403 Forbidden.
  - Hash khớp → trả file với `FileResponse`.
- **`file_management/urls.py`**: URL pattern mới `download/<str:file_type>/<int:pk>/`.
- **`base/urls.py`**: Wire vào `file-management/` path.
- **`FileInputNoDownload` widget**: Widget Unfold tùy chỉnh — ẩn nút download, ẩn clear checkbox, hỗ trợ `accept` attribute cho từng loại file.
- **Template** `file_management/widgets/clearable_file_input_no_download.html`: Template widget Unfold-styled, hiển thị tên file dạng plain text.
- **`BaseFileAdmin`**: Admin đầy đủ tính năng:
  - `verified_download` smart button: nút xanh (OK), badge lỗi `gpp_bad` (bị sửa đổi), badge lỗi `error` (không đọc được).
  - `integrity_status`, `hash_display`, `current_hash_display` readonly fields.
  - `get_fieldsets` động: auto → chỉ `verified_download`; upload → `file + verified_download`; mới → chỉ `file`.
  - `change_view` override: hiển thị Django message banner khi phát hiện tamper hoặc file missing.
  - `get_readonly_fields` thêm `file` vào readonly khi `method='auto'`.
  - `accepted_file_types` per subclass (Excel: `.xlsx,.xls,.csv`, PDF: `.pdf`).
- **`ExcelFileAdmin`** và **`PDFFileAdmin`**: Kế thừa `BaseFileAdmin` với `file_type` và `accepted_file_types` cụ thể.
- **Migration**: `0004_excelfile_hash_pdffile_hash.py` — thêm field `hash` (CharField 64, nullable) cho `ExcelFile` và `PDFFile`.
- **Bản dịch tiếng Việt**: Tất cả chuỗi mới được dịch đầy đủ vào `locale/vi/LC_MESSAGES/django.po`, biên dịch `.mo`.

### Sửa Lỗi
- **XSS trong `status_view`/`method_view`**: Đổi từ `format_html(f'<span...{var}>')` sang `format_html('<span class="...">{}</span>', escaped_value)` — an toàn OWASP.
- **Stale hash cache**: Xóa bỏ `_file_hash_cache` trên admin instance (singleton) gây stale data sau upload. Luôn đọc hash mới nhất.
- **`FieldFile.open()` với Celery**: File tạo bởi task `wn_import_export` bị exception khi gọi `FieldFile.open()` trên file đã đóng. Fix: dùng `storage.open(file_field.name, 'rb')` trực tiếp.
- **Legacy enrollment**: Records cũ không có hash (hash=NULL) không bị chặn tải về — tự động tính và lưu hash lần đầu.

## 4. Tương Thích Và Rủi Ro

- **Tương thích ngược**: Hoàn toàn tương thích ngược. Field `hash` nullable — records cũ (hash=NULL) tự động được enroll khi download lần đầu.
- **Migration**: Không phá vỡ schema. Chỉ thêm 1 nullable column, không cần dữ liệu migration.
- **Celery tasks**: Fix đảm bảo `wn_import_export` tasks tạo file không bị exception.
- **Rủi ro còn lại**: Files được tạo bởi external tools (ngoài Django ORM) không có hash → sẽ tự động enrolled khi download lần đầu. Nếu file bị xóa khỏi storage trực tiếp mà không qua ORM, download sẽ trả 403.

## 6. Hướng Dẫn Nâng Cấp

**Bước triển khai:**
1. Upgrade package: `uv add django-whiteneuron==0.3.0` (hoặc pip install).
2. Chạy `python manage.py migrate` để áp dụng migration `0004_excelfile_hash_pdffile_hash`.
3. Không cần thay đổi gì thêm — tự động backward compatible.
4. Legacy files sẽ được enrolled (hash tính lần đầu) khi user download lần đầu.

**Điều kiện rollback:**
- Rollback về `0.2.48.5`: `uv add django-whiteneuron==0.2.48.5` + `python manage.py migrate file_management 0003` (reverse migration để xóa field `hash`).
- Không có dữ liệu bị mất khi rollback (field `hash` nullable, xóa column an toàn).
