# Release v0.3.4.3 — Configurable Meta Fieldset Collapse

## 1. Summary
- **Version**: v0.3.4.3 (2026-05-03)
- **Release type**: Patch
- **Scope**: ModelAdmin UX improvement — configurable meta fieldset behavior

## 2. Main Changes
- Added `ModelAdmin.meta_class_in_fieldsets` attribute to control Meta fieldset collapse state

## 3. New Features
- **Configurable Meta Fieldset**: Previously, the Meta fieldset (containing created_at, created_by, updated_at, updated_by, is_hidden, is_deleted) was always collapsed in change forms. Now admins can set `meta_class_in_fieldsets = 't'` on any ModelAdmin subclass to keep it expanded by default.

## 4. Compatibility & Risk
- **Compatibility**: No breaking changes; safe for all v0.3.x users
- **Risk**: Minimal — only adds a new optional attribute with sensible default (`'collapse'`)
- **Mitigation**: Existing deployments unaffected; opt-in via subclass override
