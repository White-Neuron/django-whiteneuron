# Release v0.2.36 — 2026-03-30

## 1. Tóm Tắt Phát Hành

- **Phiên bản:** 0.2.36
- **Mục tiêu phát hành:** Bảo mật lệnh `init_admin` — xoá password hardcode, sinh mật khẩu ngẫu nhiên mạnh, gửi email tự động, hỗ trợ reset password có kiểm soát.
- **Phạm vi thay đổi:** `base/management/commands/init_admin.py`, `base/utils.py`, `templates/admin/base/email_password.html` (xoá) — không có migration mới.

## 2. Thay Đổi Chính

- **Security — Xoá hardcode password:** Password `'wnadmin2024&'` trong `init_admin.py` được thay bằng `secrets.choice()` (CSPRNG) sinh mật khẩu 20 ký tự đảm bảo đủ chữ hoa, thường, số và ký tự đặc biệt. Không còn credential nào nằm trong source code.
- **Added — `INIT_ADMIN_PASSWORD` env var:** Nếu có, dùng password này thay vì sinh ngẫu nhiên — phù hợp môi trường staging cần password cố định biết trước.
- **Added — `INIT_ADMIN_EMAIL` env var (bắt buộc):** Email nhận mật khẩu tạm thời. Không còn hardcode `ntanhtm@gmail.com`. Nếu thiếu, lệnh báo lỗi rõ ràng và không tạo user.
- **Added — Gửi email tự động:** Khi password là random, hệ thống gửi email qua `send_email_login()` dùng template chuẩn của dự án (chữ ký công ty, link đăng nhập). Password cũng in ra stdout một lần.
- **Added — Flag `--reset-password`:** Cho phép đặt lại mật khẩu admin đã tồn tại. Mặc định không tự reset để tránh lock-out trong CI/CD. Khi reset cũng gửi email với subject phân biệt.
- **Fixed — Double DB write:** Trước đây tạo user tốn 2 lần hash + 2 lần ghi DB (`placeholder` → real password). Giờ `_apply_password()` tự detect `user.pk` để quyết định `save()` hay `save(update_fields=['password'])` — chỉ 1 lần ghi khi tạo mới.
- **Fixed — Validate email trước khi ghi DB:** Nếu thiếu email, lệnh raise `ValueError` trước khi `set_password()` — tránh trường hợp password bị đổi nhưng email gửi thất bại.
- **Improved — `send_email_login()` nhận `is_reset`:** Subject email thay đổi theo ngữ cảnh: `"Thông tin đăng nhập hệ thống"` (tạo mới) vs `"Đặt lại mật khẩu hệ thống"` (reset).
- **Removed — `templates/admin/base/email_password.html`:** File HTML không được dùng ở bất kỳ đâu (chỉ tồn tại trong comment dead code). `send_email_login()` dùng `TEMPLATE` Python string trong `utils.py`.

## 3. Cấu Hình `.env` Cần Thêm

```env
# Bắt buộc khi dùng init_admin
INIT_ADMIN_EMAIL=your-admin@domain.com

# Tùy chọn — nếu không đặt, hệ thống sinh ngẫu nhiên và gửi email
INIT_ADMIN_PASSWORD=YourStrongPass!23
```

## 4. Cách Dùng

```bash
# Lần đầu — tạo mới admin
python manage.py init_admin

# Admin đã tồn tại, muốn reset mật khẩu
python manage.py init_admin --reset-password
```

## 5. Tương Thích Và Rủi Ro

- **Không breaking change** với code hiện tại — `send_email_login()` thêm param `is_reset=False` (default), tất cả caller cũ không bị ảnh hưởng.
- **Cần thêm `INIT_ADMIN_EMAIL` vào `.env`** trước khi chạy `init_admin` — nếu thiếu lệnh báo lỗi rõ ràng, không crash ngầm.
- Không có migration mới.

## 6. Xác Thực Đã Thực Hiện

- **Review logic:** Validate email trước DB write, single save path, CSPRNG password generation.
- **Makemigrations:** No changes detected.
