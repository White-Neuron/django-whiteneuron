# Release v0.2.28

Date: 2026-03-28

## Scope
- Tối ưu giao diện thẻ User trong grid view: chuyển trạng thái và vai trò từ text badge sang icon, gọn và trực quan hơn.
- Chuẩn hóa `verbose_name` cho các trường trong `Notification` và `NotificationConfig` để hiển thị admin rõ ràng hơn.
- Bổ sung migration cho các thay đổi schema-level metadata (AlterField).
- Cập nhật bản dịch tiếng Việt tương ứng và biên dịch lại `.mo`.
- Nâng phiên bản package lên `0.2.28`.

## Changes

### pyproject.toml
- Bump phiên bản từ `0.2.27` -> `0.2.28`.

### whiteneuron/base/admin.py
- Trong `UserAdmin.grid_item_header`:
  - Đổi role badge thành icon Material Symbols có tooltip:
    - `Superuser` -> `admin_panel_settings`
    - `Staff` -> `badge`
    - `Bot` -> `smart_toy`
  - Đổi trạng thái `Active/Inactive` từ badge text sang icon:
    - `Active` -> `check_circle`
    - `Inactive` -> `cancel`

### whiteneuron/notification/models.py
- Bổ sung `verbose_name` cho các field trong `NotificationConfig`:
  - `model`, `event_type`, `send_to_admin`, `send_to_user`, `send_to_group_user`, `group_users`, `users`.
- Bổ sung `verbose_name` cho các field trong `Notification`:
  - `obj_link` -> `Object Link`
  - `flag` -> `Notification Type`
  - `action` -> `Action Type`
  - `created_at` -> `Created At`

### whiteneuron/notification/migrations/0012_alter_notification_action_and_more.py
- Thêm migration `AlterField` đồng bộ các thay đổi `verbose_name` và cấu hình field ở app `notification`.

### whiteneuron/locale/vi/LC_MESSAGES
- Cập nhật `django.po` với các msgid mới liên quan `Notification`/`NotificationConfig`.
- Biên dịch lại `django.mo`.

### whiteneuron/static/base/css/styles.css
- Cập nhật CSS build output sau thay đổi class UI trong admin card.

## Compatibility and Risk
- Không có thay đổi dữ liệu mang tính phá hủy; migration chỉ gồm `AlterField` metadata.
- Thay đổi giao diện chỉ tác động phần hiển thị admin, không ảnh hưởng business logic.
- Rủi ro thấp; cần chú ý đồng bộ static để tránh mismatch CSS cache ở môi trường production.

## Upgrade Guidance
- Nâng package lên `0.2.28`.
- Chạy `python manage.py migrate` để áp dụng migration `notification/0012`.
- Thu gom và deploy static mới.
- Xóa cache trình duyệt (nếu cần) để thấy UI icon mới trong admin grid.

## Rollback
- Roll back về tag `v0.2.27` nếu phát sinh sự cố.
- Có thể migrate ngược về `notification 0011` nếu cần hoàn tác migration metadata:
  - `python manage.py migrate notification 0011`
