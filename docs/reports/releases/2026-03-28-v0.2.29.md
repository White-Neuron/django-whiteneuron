## 1. Tóm Tắt Phát Hành

- **Phiên bản:** 0.2.28 → 0.2.29
- **Mục tiêu:** Sửa lỗi `NotificationAdmin` — nút View Linked Object bị 403, action không được route đúng, và các trường hiển thị admin thiếu khai báo `readonly_fields`.
- **Phạm vi thay đổi:** Hẹp — chỉ ảnh hưởng module `notification/admin.py` và locale tiếng Việt.

---

## 2. Thay Đổi Chính

### `whiteneuron/notification/admin.py`

- **Chuyển action từ `actions_submit_line` sang `actions_detail`:** Submit-line action bắt buộc POST qua Django change view, bị chặn bởi `has_change_permission = False`. Chuyển sang `actions_detail` (hiển thị trên trang chi tiết, route riêng qua URL) giải quyết triệt để lỗi 403.
- **Refactor `view_obj_link`:** Action nhận `object_id: int` thay vì `obj`, thêm `url_path="view-obj-link"` để có route riêng. Xử lý graceful khi `obj_link` rỗng — hiển thị message và redirect về trang trước thay vì lỗi.
- **Refactor `has_view_obj_link_permission`:** Nhận `object_id` trực tiếp, bỏ fallback `getattr(obj, "pk", obj)` phức tạp, logic đơn giản và rõ ràng hơn.

### `whiteneuron/locale/vi/LC_MESSAGES/django.po` / `.mo`

- Cập nhật chuỗi dịch: `"View"` → `"View Linked Object"` / `"Xem đối tượng được liên kết"`.
- Thêm chuỗi mới: `"No object link available for this notification."` → `"Không có liên kết đối tượng nào có sẵn cho thông báo này."`.
- Cập nhật số dòng tham chiếu sau khi refactor file admin.
