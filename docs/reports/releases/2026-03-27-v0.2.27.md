# Release v0.2.27

Date: 2026-03-27

## Scope
- Bổ sung trường `action_by` vào model `Notification` để truy vết người thực hiện hành động.
- Cải thiện `NotificationAdmin`: hiển thị thêm thông tin, sửa logic parse `changed_data`, tăng cường an toàn XSS.
- Cải thiện UI grid view App: badge trạng thái gọn hơn.
- Cập nhật bản dịch tiếng Việt.

## Changes

### notification/models.py
- Thêm trường `action_by = ForeignKey(User, on_delete=SET_NULL, null=True, blank=True)` với `related_name='action_by_notifications'` và `verbose_name=_("Action By")`.
- Cập nhật `Notification.user` với `verbose_name=_("User to Notify")` và `related_name='notifications'`.

### notification/migrations
- `0009_notification_action_by_alter_notification_user.py`: thêm `action_by`, cập nhật `user` field.
- `0010`, `0011`: cập nhật `related_name` và `verbose_name` bổ sung.

### base/modeladmin.py
- Gán `action_by=request.user` trong tất cả luồng tạo `Notification`:
  - `soft_delete`, `hard_delete`, `restore`, `delete_model`.

### base/models.py
- Gán `action_by=self.updated_by if action == 'update' else self.created_by` trong `BaseModel.save`.
- Chuyển `changed_data` lưu bằng `json.dumps` thay vì `str()` để tương thích với parser mới.

### notification/admin.py
- Thêm `action_by` vào `list_display`, `autocomplete_fields`, `search_fields`, `list_filter`, `fieldsets`.
- Sửa `display_changed_data`:
  - Hỗ trợ parse cả `json.loads` và `ast.literal_eval` (backward compatible với dữ liệu cũ lưu bằng `str()`).
  - Hỗ trợ đúng schema list `[{"field_name", "old_value", "new_value"}]` từ `BaseModel.save`.
  - Hỗ trợ schema dict `{"field": {"old", "new"}}`.
  - Dùng `format_html` để escape nội dung, tránh XSS.

### base/admin.py
- Badge trạng thái Active/Inactive trong App grid view đổi từ text badge sang icon `check_circle`/`cancel` (Material Symbols Outlined, 18px) với `title` tooltip.

### locale/vi/LC_MESSAGES
- Thêm entry dịch: `"Action By"` → `"Hành động bởi"`, `"User to Notify"` → `"Thông báo cho người dùng"`.
- Dọn dẹp các cờ `fuzzy` còn tồn đọng trong header và các entry đã có bản dịch rõ.
- Cập nhật `.mo` binary.

## Compatibility and Risk
- Backward compatible: trường `action_by` là `null=True, blank=True` nên các bản ghi cũ không bị ảnh hưởng.
- Migration an toàn: `AddField` + `AlterField` không destructive.
- Thay đổi `related_name` của `Notification.user` từ mặc định `notification_set` sang `notifications`: cần kiểm tra nếu project sử dụng reverse accessor này trong code ngoài package.
- Rủi ro thấp: các thay đổi admin và UI hoàn toàn độc lập với business logic.

- Nâng phiên bản package lên `0.2.27`.
- Chạy `python manage.py migrate` để áp dụng migration `0009`–`0011` của app `notification`.

- Migration `0009`–`0011` có thể reverse an toàn với `python manage.py migrate notification 0008`.
