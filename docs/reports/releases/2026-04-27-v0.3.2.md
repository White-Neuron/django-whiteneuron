# Release v0.3.2 — VisitProfile-based Activity Tracking & Dashboard Analytics Expansion

## 1. Summary
- **Version**: `v0.3.2`
- **Release Date**: 2026-04-27
- **Scope**: New activity tracking architecture (VisitProfile), anonymous visitor analytics, dashboard expansion, admin UI improvements

## 2. Main Changes
- **Activity tracking refactored** — `UserActivity` no longer stores `ip_address`/`user_agent` directly; now linked via FK to new `VisitProfile` model for deduplication and cross-request session tracking.
- **Anonymous visitor tracking added** — new `AnonymousActivity` model tracks unauthenticated requests with full path, method, status code, and timelapse data.
- **Middleware enhanced** — `UserActivityMiddleware` now creates `VisitProfile` entries for both authenticated and anonymous users; race condition handled via `IntegrityError` retry.

## 3. New Features
- **Dashboard analytics expansion** — KPI cards now show success rate percentages; 28-day chart expanded from 3 to 6 datasets (user avg/success/error + anonymous avg/success/error).
- **Admin views** — new `VisitProfileAdmin`, `AnonymousActivityAdmin`; inline activity viewing from User detail page via `BaseActivityInline`.
- **Cleanup command** — `cleanup_old_activities.py` management command for purging old activities and orphan profiles (default 90 days, supports `--dry-run`).
- **Sidebar navigation** — new "Anonymous activity" and "Visit profiles" links with badge counts.

## 4. Compatibility & Risk
- **Compatibility**: Django >= 5.2.13, Python >= 3.11 — no breaking changes to existing APIs.
- **Risk**: Low. Migration chain (0020→0026) backfills `VisitProfile` from existing `UserActivity` data; safe reverse migrations provided.
- **Mitigation**: Run `python manage.py migrate` after deploy; verify `visit_profiles` table populated correctly.

## 6. Upgrade Guide
1. Update package: `pip install --upgrade django-whiteneuron` or `uv upgrade django-whiteneuron`
2. Run migrations: `python manage.py migrate base`
3. Verify dashboard analytics display correctly for both user and anonymous activities

**Rollback**: Revert to previous version; old `UserActivity` records with `ip_address`/`user_agent` fields will be null after migration (data preserved in `VisitProfile`).
