# Release Notes — v0.3.4.2

**Date**: 2026-05-03  
**Version**: 0.3.4.2  
**Branch**: main2.0  

## Summary

Patch release fixing `duplicate_objects` action to support models with `title` field (e.g. `App.title`) in addition to `name`. No schema changes; safe upgrade for all v0.3.x users.

## Main Changes

### Fixed
- **`ModelAdmin.duplicate_objects()`** (`whiteneuron/base/modeladmin.py`) — previously only copied objects with a `name` field, ignoring the `title` field used by models like `App`. Now checks both fields:
  - If object has `name` → appends `(Copy)` to `name`
  - Else if object has `title` → appends `(Copy)` to `title`
  - Otherwise → duplicates without renaming (graceful fallback)
