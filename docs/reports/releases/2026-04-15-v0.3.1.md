# Release v0.3.1 — 2026-04-15

## 1. Tóm Tắt Phát Hành
- **Phiên bản**: 0.3.1
- **Mục tiêu phát hành**: Bổ sung tính năng Announcement System cho admin header — cho phép dự án con hiển thị thông báo quan trọng (release notes, cập nhật hệ thống) trực tiếp trên header với nội dung lazy-loaded và giao diện modal.
- **Phạm vi thay đổi**: `base/settings.py`, `base/sites.py`, `base/utils.py`, `base/views.py`, `base/urls.py`, 2 template mới (`announcement.html`, `userlinks.html`), `locale/vi`, Tailwind rebuild.

## 2. Thay Đổi Chính

### Added
- **`base/settings.py`** — ba setting mới để cấu hình hệ thống announcement:
  - `VERSION` — phiên bản ứng dụng (lấy từ biến môi trường `VERSION`, mặc định `"0.1.0"`).
  - `ANNOUNCEMENT_CALLBACK` — đường dẫn Python tới hàm trả về context announcement (mặc định: `"whiteneuron.base.utils.announcement_callback"`).
  - `ANNOUNCEMENT_CONTENT_HTML_FILE` — template name hoặc absolute `Path` tới file HTML nội dung thông báo (mặc định: `"announcement_content.html"`).
- **`base/sites.py`** — override `each_context()` trong `BaseAdminSite`: tự động import và gọi `ANNOUNCEMENT_CALLBACK`; inject key `announcement` vào context mọi trang admin; lỗi trong callback bị bắt silently.
- **`base/utils.py`** — `announcement_callback(request)` mặc định: kiểm tra `ANNOUNCEMENT_CONTENT_HTML_FILE`, trả về dict `{title, version, badge}` với `format_lazy`/`gettext_lazy` cho i18n đầy đủ.
- **`base/views.py`** — `get_announcement_content(request)` view: render `ANNOUNCEMENT_CONTENT_HTML_FILE` qua `render_to_string`; trả về `HttpResponse` với nội dung HTML.
- **`base/urls.py`** — endpoint `/announcement/` wired tới `get_announcement_content`.
- **`templates/base/announcement.html`** — modal dialog (DaisyUI `.ui-modal`) với:
  - Header: icon `campaign`, tiêu đề, version, badge.
  - Scrollable content area: lazy-load HTML qua `fetch()` khi mở lần đầu.
  - Footer: nút "Got it" với `{% trans %}`.
  - JS tự quản lý trạng thái `loaded` để tránh fetch lại mỗi lần mở.
- **`templates/unfold/helpers/userlinks.html`** — nút thông báo trên header:
  - Hiển thị khi `{{ announcement }}` có trong context.
  - Animation `wn-glow-pulse` (glow cam) và `wn-flame-icon` (flame breathing).
  - `onclick` mở modal `wn-announcement-modal`.
  - Hiển thị badge số nếu có `announcement.badge`.

### Updated
- **`locale/vi/LC_MESSAGES/django.po|.mo`** — bổ sung và cập nhật bản dịch tiếng Việt cho các chuỗi i18n mới liên quan đến announcement.
- **`static/base/css/styles.css`** — Tailwind rebuild để include utility classes mới dùng trong các template announcement.

## 4. Tính Năng Mới

**Announcement System** là hệ thống thông báo pluggable cho admin:

1. **Cấu hình trong dự án con**:
   ```python
   # settings.py của dự án con
   VERSION = "1.5.2"
   ANNOUNCEMENT_CALLBACK = "myapp.utils.get_announcement"
   ANNOUNCEMENT_CONTENT_HTML_FILE = BASE_DIR / "templates" / "announcement_content.html"
   ```

2. **Callback tùy chỉnh**:
   ```python
   def get_announcement(request):
       return {
           "title": "Phiên bản 1.5.2 — Tính năng mới",
           "version": "1.5.2",
           "badge": "MỚI",
       }
   ```

3. **File nội dung HTML** (`announcement_content.html`) — nội dung HTML tùy ý (release notes, hướng dẫn...) sẽ được render vào phần scrollable của modal.

4. **Tắt announcement**: đặt `ANNOUNCEMENT_CONTENT_HTML_FILE = None` hoặc `ANNOUNCEMENT_CALLBACK = None`.

## 5. Tương Thích Và Rủi Ro
- **Tương thích ngược**: Đầy đủ. Các dự án không cấu hình `ANNOUNCEMENT_CALLBACK` hoặc `ANNOUNCEMENT_CONTENT_HTML_FILE` sẽ không hiển thị gì (callback silently fail).
- **Breaking change**: Không có.
- **Rủi ro**: Thấp. Lỗi trong `ANNOUNCEMENT_CALLBACK` bị bắt bởi `try/except Exception` — không ảnh hưởng tới giao diện admin.
- **Template override**: Dự án con đã override `userlinks.html` cần merge thay đổi thủ công.
