# v0.2.43 — 2026-04-01

## Feature: Feedback System — DaisyUI modals, anti-spam cooldown, i18n, security hardening

### Overview

Nâng cấp toàn diện hệ thống feedback trong Django Admin. Thay thế toàn bộ `prompt()`/`alert()` native bằng DaisyUI `<dialog>` modals, thêm chống spam server-side với thông báo động, áp dụng i18n đầy đủ, và vá các lỗ hổng bảo mật.

---

### Changes

#### `feedbacks/urls.py` — Security & anti-spam

- **Security**: Thay `@csrf_exempt` bằng `@require_POST` — CSRF middleware tự xử lý header `X-CSRFToken` từ frontend, không cần bypass; loại bỏ kiểm tra method thủ công.
- **Security**: Auth check được chuyển lên đầu hàm, trước khi parse request body — tránh xử lý input không cần thiết với unauthenticated requests.
- **Removed**: Redundant `User.objects.get(pk=request.user.pk)` — dùng `request.user` trực tiếp.
- **Added**: Server-side cooldown 60 giây per user (toàn cục, không phân biệt đối tượng) — trả về HTTP 429 với số giây còn lại tính động: `remaining = FEEDBACK_COOLDOWN_SECONDS - elapsed`.
- **Added**: `FEEDBACK_COOLDOWN_SECONDS = 60` constant — dùng chung với admin.py.

#### `feedbacks/admin.py` — Code quality & null safety

- **Fixed**: Import order — xoá `from unfold.decorators import display` (unused), gộp `from django.urls import reverse` vào block Django imports, chuyển `from .urls import FEEDBACK_COOLDOWN_SECONDS` từ cuối file lên đầu.
- **Fixed**: Null safety — `self.get_object(request, object_id)` có thể trả `None` nếu object không tồn tại; thêm guard `if obj is None: return super().changeform_view(...)` tránh `AttributeError`.
- **Added**: `FeedbackBaseAdmin.render_change_form()` truyền `feedback_cooldown_ms = FEEDBACK_COOLDOWN_SECONDS * 1000` vào template context từ server thay vì hardcode ở frontend.

#### `templates/admin/feedbacks/feedback_change_form.html` — DaisyUI modal

- **Added**: DaisyUI `<dialog>` prompt modal — nhập note trước khi "Mark as Resolved", thay thế `prompt()` native.
- **Added**: DaisyUI `<dialog>` result modal — hiển thị kết quả success/error.
- **Fixed**: OK button trong result modal bọc trong `<form method="dialog">` — đảm bảo `<dialog>.close()` được gọi đúng cách, `close` event kích hoạt `location.reload()`.
- **Fixed**: Modal sequencing — `promptModal.addEventListener('close', oneTimeHandler)` trước khi `resultModal.showModal()`, tránh overlap animation.
- **Added**: `{% trans %}` cho toàn bộ text trong template; JS strings dùng `{% trans "..." as var %}` + `{{ var|escapejs }}`.
- **Fixed**: Backdrop buttons — text `close` rỗng (dùng `aria-label`), tránh flash text trong closing animation.

#### `templates/admin/submit_line.html` — DaisyUI modal + UX

- **Added**: "Phản hồi" button trong submit row của mọi model admin kế thừa `FeedbackBaseAdmin`.
- **Added**: DaisyUI prompt modal với `<textarea>` multi-line input và loading spinner trong nút Gửi.
- **Added**: DaisyUI result modal hiển thị thông báo server (bao gồm nội dung cooldown message).
- **Fixed**: Buttons thiếu `type="button"` — tránh submit admin form ngoài khi click Cancel/Gửi/OK.
- **Fixed**: `document.body.appendChild(promptModal/resultModal)` — di chuyển dialog ra khỏi submit row DOM, tránh flash nội dung ở cuối trang trong closing animation.
- **Fixed**: Spinner ẩn ngay sau fetch hoàn thành (`resetBtn()` luôn gọi dù success hay error) — không giữ UI trong loading state; cooldown thực thi ở server.
- **Fixed**: `{{ object_id|escapejs }}` trong JSON body — đúng escaping convention cho JS string interpolation.
- **Added**: `{% trans %}` / `{% blocktrans with name=subtitle %}` cho toàn bộ text.
- **Added**: `getCookie('csrftoken')` helper — gửi CSRF token qua `X-CSRFToken` header.

#### `base/modeladmin.py` — i18n

- **Added**: `gettext_lazy as _` cho toàn bộ hardcoded strings (12 strings): `short_description`, `message_user`, `search_help_text`, v.v.
- **Fixed**: f-string trong `_()` → format bằng `%` operator để `makemessages` extract đúng.

#### `locale/vi/LC_MESSAGES/django.po` — Translations

- Thêm dịch tiếng Việt cho toàn bộ strings mới từ feedback system và modeladmin.

### Security Notes

| Issue | Before | After |
|-------|--------|-------|
| CSRF bypass | `@csrf_exempt` | `@require_POST` (CSRF enforced) |
| Auth check order | After body parse | Before body parse |
| Redundant DB query | `User.objects.get(pk=...)` | `request.user` trực tiếp |
| Null dereference | `get_object()` result used without check | `if obj is None` guard |

---

### Upgrade Notes

- Không có breaking changes.
- Không có migration mới.
- Các model admin muốn hiển thị nút "Phản hồi" cần kế thừa `FeedbackBaseAdmin` (thay vì `ModelAdmin`).
- Chạy `compilemessages` sau khi deploy nếu dùng locale `vi`.
