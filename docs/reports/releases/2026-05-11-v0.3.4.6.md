# Release v0.3.4.6 (2026-05-11)

## Summary

Bugfix release addressing critical crashes, data integrity issues, and code quality improvements across the dashboard, notification, middleware, and admin modules.

## Main Changes

### Added
- None

### Fixed
- **Dashboard month arithmetic crash** (`dashboard/views.py`): `start.replace(month=start.month+1)` raised `ValueError` on December; replaced with `(start + timedelta(days=32)).replace(day=1)` idiom that handles all edge cases. Similarly fixed month underflow for January.
- **Notification scope bug** (`notification/models.py`): `mark_as_read_all()` / `mark_as_unread_all()` updated ALL notifications in the database regardless of user; now scoped to specific user via `cls.objects.filter(user=user).update(...)`.
- **Middleware typo** (`base/middleware.py:241`): `'startwith'` typo prevented `/ws/` WebSocket paths from being excluded from activity logging and rate limiting. Fixed to `'startswith'`.
- **ModelAdmin filter mutation** (`base/modeladmin.py:237`): `self.list_filter` was mutated across requests, causing `FieldSelectionFilter` to accumulate on every call. Added guard condition.
- **Password save order** (`base/admin.py:343`): User admin password flow refactored with `is_new` flag so `set_password()` is called after `super().save_model()` without relying on stale `obj.pk`.
- **Redirect URL pattern** (`notification/admin.py:102`): `NotificationAdmin.view_obj_link()` used malformed string URL for redirect; now uses `reverse('admin:...')`.
- **HTML typo** (`base/modeladmin.py:405`): `action_buttoms` → `action_buttons` in action buttons template (both ID and CSS class).

### Improved
- **Badge callback return types** (`base/utils.py`): Unified to always return string (`f"{c}"`), removed redundant zero checks since `f"{0}" == "0"`.
- **Exception handling**: Replaced bare `except:` with `except Exception:` across 6 files (`models.py`, `modeladmin.py`, `notification/admin.py`, `filters.py`).

## Compatibility Notes

- **No breaking changes**. All fixes are backward-compatible.
- `mark_as_read_all()` / `mark_as_unread_all()` changed from instance methods to classmethods — if any code calls these on an instance, it will break. However, grep found no callers in the codebase.
- Safe for all v0.3.x users.

## Upgrade Guidance

1. Install: `pip install django-whiteneuron==0.3.4.6` or `uv add django-whiteneuron==0.3.4.6`
2. No migrations required — no schema changes in this release.
3. No manual configuration changes needed.

## Rollback

Safe to revert to v0.3.4.5:
```bash
pip install django-whiteneuron==0.3.4.5
```
No data loss or migration rollback needed.
