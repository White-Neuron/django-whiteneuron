# Release v0.2.48.3 — 2026-04-10

## Summary

Patch release improving the notification admin display and fixing search field verbose name resolution.
No database migrations. No breaking changes.

---

## Changes

### Added
- `NotificationAdmin.get_components()` helper — parses `obj_link` URL to resolve `app_label`, `model_verbose`, `object_id`, and object `str()` via `ContentType` lookup.

### Improved
- **`NotificationAdmin.display_title`** — completely rewritten to render a structured badge row:
  `[Model] [#ID] [Object name] [Action] by [Alias] [@username]`
  using correct DaisyUI v5 `ui-badge ui-badge-xs` classes with color mapping:
  - `create` → `ui-badge-success`
  - `update` → `ui-badge-info`
  - `delete` → `ui-badge-error`
  - `restore` → `ui-badge-warning`

### Fixed
- **`obj_link` regex** — previous pattern `/admin/([^/]+)/([^/]+)/(\d+)/` failed to match URLs without the `/admin/` prefix (e.g. `/techniques/technique/58316/change/`).
  New pattern: `/([^/]+)/([^/]+)/(\d+)/change/` — works for both prefixed and non-prefixed paths.
- **`get_verbose_name_field`** in `modeladmin.py` — previously called `model._meta.get_field('user__username')` which raises `FieldDoesNotExist`. Now traverses each `__`-separated segment, following `related_model`, returning joined verbose names (e.g. `"user / username"`). All search fields in `search_help_text` now resolve correctly.
