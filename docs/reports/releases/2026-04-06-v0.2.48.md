# Release v0.2.48 — 2026-04-06

## Tóm tắt

Phiên bản này bổ sung tính năng **Change History Panel** — một modal xem lịch sử thay đổi trực tiếp trên change-form của Django Admin. Panel hiển thị diff `cũ → mới` cho từng trường, được bảo vệ bằng permission gate, và tích hợp hoàn toàn với hệ thống `Notification` đã có.

---

## Thay đổi chính

### Feature: Change History Panel

- **Added**: Floating button "Lịch sử" (`type="button"`, `fixed bottom-20 right-6`) hiển thị trên mọi change-form. Bấm mở modal DaisyUI với danh sách 50 bản ghi `Notification` gần nhất được lọc theo `obj_link` của object đang xem.
- **Added**: Mỗi dòng lịch sử hiển thị: badge action (Create/Update/Delete/Restore với màu tương ứng), avatar + username của người thực hiện, timestamp, và danh sách field thay đổi theo dạng `old_value (gạch ngang) → new_value`.
- **Added**: `has_view_history_permission(request, obj=None)` trên `ModelAdmin` — chỉ superuser hoặc user có `change_permission` mới thấy panel. Read-only staff không thấy.
- **Added**: `show_history = True` class attribute — subclass có thể tắt per-model bằng `show_history = False`.
- **Added**: `wn_change_form.html` — base change-form template, override `{% block object-tools-items %}` rỗng để ẩn nút Lịch sử mặc định của UNFOLD (tránh duplicate).
- **Added**: `wn_history_panel.html` — partial template cho modal: header cố định, scrollable list, diff display.

### Fixes

- **Fixed**: `<button type="button">` trên trigger — tránh accidental form submit khi bấm nút History.
- **Fixed**: Đóng modal bằng `dialog.close()` via JS thay vì `<form method="dialog">` lồng bên trong admin form (vi phạm HTML spec, khiến button bị đẩy ra ngoài DOM).
- **Fixed**: DaisyUI v5 prefix `ui-` áp dụng đúng cho `ui-modal`, `ui-modal-box`, `ui-modal-backdrop`, `ui-btn` — sửa modal hiển thị lỗi/vô hình.
- **Fixed**: UNFOLD built-in "Lịch sử" button bị ẩn đúng cách qua block override.

### Improvements

- **Improved**: `import ast` và `reverse` chuyển lên module-level imports trong `modeladmin.py` (thay vì import trong từng request).
- **Improved**: Toàn bộ chuỗi trong panel dùng `{% trans %}` — hỗ trợ đầy đủ i18n.
- **Improved**: `change_list.html` — trạng thái phân trang (`p=...`) giờ được lưu vào `sessionStorage` cùng với filter state; quay lại từ change form sẽ restore đúng trang đang xem thay vì nhảy về trang 1.

## Khả năng tương thích

- Không có migration mới.
- Không thay đổi API công khai — `show_history` và `has_view_history_permission` là additive.
- Yêu cầu `Notification` model có trường `changed_data`, `action_by`, `obj_link`, `action`, `created_at` — đã có từ v0.2.22+.
- DaisyUI v5 với tiền tố `ui-` bắt buộc.

## Hướng dẫn nâng cấp

```bash
uv add django-whiteneuron==0.2.48
```

Không cần thao tác thêm. Nếu subclass ModelAdmin của bạn có custom `change_form_template` trỏ thẳng tới `admin/change_form.html`, hãy đổi thành `admin/wn_change_form.html` hoặc extend từ đó để nhận panel.

Để tắt panel cho một model cụ thể:

```python
class MyModelAdmin(ModelAdmin):
    show_history = False
```
