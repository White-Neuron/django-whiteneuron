# Release v0.3.4.11 (2026-06-18)

## Summary

Security patch: Upgrade `cryptography` ≥48.0.1 and relax `pyopenssl` constraint to resolve CVE with CVSS 7.5 affecting versions >=0.5.0,<48.0.1. All 92 packages scanned via OSV.dev API — zero known vulnerabilities remaining.

## Main Changes

### Updated
- **cryptography**: minimum version bumped from `>=46.0.7` to `>=48.0.1` — resolves CVE with CVSS 7.5 (affects versions >=0.5.0,<48.0.1). Resolved to cryptography==49.0.0 by uv lock.
- **pyopenssl**: pinned version changed from `==26.0.0` to `>=26.2.0` — required for compatibility with cryptography ≥48.0.1 (pyopenssl 26.3.0 resolved).

## Files Changed

| File | Change |
|------|--------|
| `pyproject.toml` | version bump, cryptography bump, pyopenssl constraint relaxed |
| `uv.lock` | Resolved: cryptography v46.0.7 → v49.0.0, pyopenssl v26.0.0 → v26.3.0 |
| `CHANGELOG.md` | Added v0.3.4.11 entry |
| `README.md` | Updated version and changelog section |

## Compatibility Notes

- No breaking changes; safe for all v0.3.x users.
- `pyopenssl` constraint relaxed from exact pin (`==26.0.0`) to minimum version (`>=26.2.0`) — allows uv resolver to pick compatible versions with newer cryptography releases.
- All transitive dependencies remain at latest stable versions.
