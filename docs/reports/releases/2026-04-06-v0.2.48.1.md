# v0.2.48.1 — Patch Release (2026-04-06)

## Fix: Duplicate history entries in change history panel

### Problem
When multiple superusers exist in the database, the history panel on the change-form displayed duplicate entries for every save event. Root cause: `Notification` records are created **once per superuser** on each save, so querying all notifications for an object without filtering by user returned N rows per event (where N = number of superusers).

### Solution
Filter the history query by a single reference user:
- If `request.user` is a superuser → use `request.user` directly (zero extra DB queries).
- Otherwise → resolve to `User.objects.filter(is_superuser=True).first()`.

This guarantees exactly one `Notification` row per event, with no dedup logic needed.

### Changed files
- `whiteneuron/base/modeladmin.py` — `changeform_view`: replaced `filter(obj_link=obj_link)` with `filter(obj_link=obj_link, user=ref_user)` using optimized `ref_user` resolution.
